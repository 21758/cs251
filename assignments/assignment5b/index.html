In the final two-part series of assignments for the CS 251 course, we will develop a solar system simulator. The first part of this assignment involves appliing several design patterns to implement a simple version of the simulator, which will provide the foundation that will integrate with the Android GUI in the second part of this assignment.

This second part runs as an Android application.

Primary Participants

This two-part series of assignments will implement the following set of classes.

Entity

In this assignment, Entity has become an interface. This interface, and the objects that implement it, are an implementation of the Composite pattern, as we will have a number of different types of objects that will be represented. These include:

  • Immobile Entities. Such objects may be used to model the sun and provide a fixed reference point in space around which the other objects in the system orbit. Such objects can never have a velocity.
  • Simple Entities. Such objects represent ordinary objects, such as planets, moons, asteroids, or other satellites.
  • Entities may also be aggregated into groups. These aggregates are intended to represent planets along with their satellites.

Each of these objects will be implemented by a separate class that implements the Entity interface.

The stepsimulation behavior of AggregateEntities is configurable at runtime. We do this with the Strategy Pattern. We ask that you implement two strategies: a 'Realistic' strategy, which moves each member of the aggregate independently as if it were a regular simple entity, and a 'Rigid' strategy, which moves the aggregate as if it were a single object.

Visitors

This interface describe a simple version of the [Visitor Pattern(http://en.wikipedia.org/wiki/Visitor_pattern) for objects that implement the Entity. The visitors in this assignment have additional methods to handle the various object types. In addition to the PrintVisitor and StepSimulationVisitor, in this assignment you will also need to implement a DrawVisitor, which will be used by the MainActivity to draw entities to the screen.

DrawProxy

The DrawProxy is an implementation of the Proxy Pattern. This class contains the logic necessary to draw Entities to the screen. We have provided most of the logic here.

Graduate Students

Graduate students are required to implement the stepSimulation to support multiple threads.

Provided Code

We will provide pseudocode to calculate the position and velocity of AggregateEntity objects.
