Downloaded Image Viewer (Assignment 3)

Overview of the Downloaded Image Viewer Assignment

The purpose of this assignment is to give you experience using Android to develop an app that contains a main Activity that prompts the user for a URL to an image and then uses Intents and other Activities to download the image and view it via the Gallery app.

Graduate students (or students who are taking this course for graduate credit) must implement the onCreate() method in the DownloadImageActivity class to use the Android AsyncTask framework, which is covered in this video.

Undergraduate students are welcome use the AsyncTask framework to implement this method, but may choose to use the Android "HaMeR" framework instead, which is covered in this video.

In addition to the lectures on Android we'll present in class, the following resources may be helpful in completing portions of this assignment:

  • Introduction to Android Concurrency
  • Motivations for Concurrency

Skeleton Code

Skeleton code for both the grad and ugrad versions of this assignment is available here account. Once you've setup your GitLab account you can pull this skeleton code in your repository, read it carefully, and complete the "TODO" markers.

Note that there are no automated unit tests for this assignment. Instead, you'll know the program is working when you can successfully download and view an image.

Concluding Remarks

This assignment is designed to help you get familiar with programming Android in Java. However, it is harder than the first two Java Array assignments since it involves many more "moving parts", so make sure you start early and come to office hours of you have questions or run into problems.
